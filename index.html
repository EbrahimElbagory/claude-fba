Field 03 — Contents (5)

Five skills. One loop.

Each one is a SKILL.md you drop into .claude/skills/ — carrying the hard-won gotchas, not just the happy path.

Store-AuditSKU 01

A full checkup of every listing you own: content quality score, which keywords you rank for and where, sales-rank trend, review count and rating, and how many months of stock you're sitting on. Each audit is compared to the last one, so you see exactly what moved.

Read the skill →
Review-AutomationSKU 02

Amazon lets you request one review per order, between day 5 and 30. This sends that official request for every order you've missed (my catch-up was 437 orders), then runs automatically every morning so no order is ever missed again. Can't double-send — Amazon blocks repeats and the script keeps its own log.

Read the skill →
Listing-GuardSKU 03

Edits your live listing text (titles, backend search terms) through Amazon's API — safely. It checks what Amazon allows editing on your product type, drafts the copy, shows you before/after, and changes nothing until you approve the exact wording. The old version is saved first, so undo is one command, and every change is verified after.

Read the skill →
Impact-TrackerSKU 04

Proves whether changes actually worked. Records every number BEFORE you touch anything, then logs traffic, conversion rate, units, and revenue per product every week. In 30 days you're comparing against a baseline instead of guessing — conversion moves within days, so you know early.

Read the skill →
/div>
Keyword-ResearchSKU 05

Finds the keywords worth going after: pulls what competitors rank for that you don't, expands seed terms into related searches, then sorts candidates by how cheap they are to win. Ends with a simple map — these words go in your title, these in bullets, these in backend, these into ads.

Read the skill →
Scripts/PLUMBING

The plumbing under the skills: four plain-Python files with zero dependencies to install. A small client that talks to Amazon's Seller API with just your keys, plus ready-to-run scripts for review requests, weekly tracking, and Ads API login. The cron jobs run these — and you can run them yourself, no Claude needed.

Field 05 — Declarations

Questions sellers actually ask.

Is this against Amazon's terms of service?

No. It's your own store accessed through your own self-authorized SP-API app — the same official interface every seller tool uses. Review requests go through Amazon's Solicitations API, functionally identical to the "Request a Review" button in Seller Central. Nothing is scraped.

Do I need Helium 10?

For the market layer (keyword volumes, ranks, listing quality scores, competitor intel) — yes, a paid plan; the audit uses their official MCP server. Everything SP-API-side works without it: review automation, listing edits, impact tracking, sales analytics.

What does it cost to run?

Claude Code (your existing subscription) + optionally Helium 10. The toolkit itself is free, MIT-licensed, stdlib-only Python — no servers, no pip installs. A full audit uses ~25 of your 1,000 monthly Helium 10 MCP calls.

Can it break my listings?

The harness is built so it can't do so silently: schema checked before edits are promised, every change drafted for your approval, pre-edit state backed up before any write, result verified after. Rollback = re-apply the backup.

Windows / Linux?

Scripts and skills run anywhere Python 3.9+ does. The two automation templates are macOS launchd; use cron on Linux or Task Scheduler on Windows.

What's on the roadmap?

Amazon Ads API campaign automation (CPR-sized ranking pushes — the auth client already ships in scripts/), Search Query Performance funnel analysis, multi-marketplace support. Issues and PRs welcome.
