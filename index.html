HOW IT WORKS

Two data planes, one brain, a cron.

Claude Code ──┬── Helium 10 MCP   market layer — keyword volume, ranks, LQS, CPR, competitors
              ├── SP-API (yours)  store layer  — listings, orders, solicitations, reports, catalog
              └── launchd / cron  keeps running — daily review requests, weekly impact snapshots

  skills/ ──→ .claude/skills/   scripts/ = stdlib Python, your creds, your machine

The market layer needs a paid Helium 10 plan (their official MCP). Everything on the store layer — review automation, safe listing edits, impact tracking, sales analytics — runs on a free self-authorized SP-API app you create in ~30 minutes. Setup guide →

LIVE STORE, REAL MONEY

The safety model is the feature.

Letting an agent touch a revenue-producing store only works with hard rules. These are non-negotiable in every skill:

1
Read-only by defaultA full audit writes nothing. Most sessions never mutate anything.
2
Draft → human approval → executeNo buyer-visible change ships until you approve the exact copy in chat.
3
Backup before write, verify afterPre-edit state saved per SKU; post-PATCH re-read checks for new errors. Rollback is one command.
4
Official APIs onlyYour own SP-API app, Helium 10's own MCP. Nothing scraped, no ToS games — review requests use Amazon's own mechanism.
5
Honest about limitsBullets on restricted product types, coupons, removal orders, role changes — no API path exists, and the skills say so instead of pretending.
FAQ

Questions sellers actually ask.

Is this against Amazon's terms of service?

No. It's your own store accessed through your own self-authorized SP-API app — the same official interface every seller tool uses. Review requests go through Amazon's Solicitations API, functionally identical to the "Request a Review" button in Seller Central. Nothing is scraped or automated through the website.

Do I need Helium 10?

For the market layer (keyword volumes, rank data, listing quality scores, competitor intel) — yes, a paid plan; the audit uses their official MCP server. Everything SP-API-side works without it: review automation, listing edits, impact tracking, sales analytics.

What does it cost to run?

Claude Code (your existing subscription) + optionally Helium 10. The toolkit itself is free, MIT-licensed, stdlib-only Python — no servers, no pip installs, runs on your laptop. A full store audit uses ~25 of your 1,000 monthly Helium 10 MCP calls.

Can it break my listings?

The harness is built so it can't do so silently: schema is checked before edits are promised, every change is drafted for your approval first, the pre-edit state is backed up before any write, and the result is verified after. Rollback = re-apply the backup.

Does it work on Windows / Linux?

Scripts and skills run anywhere Python 3.9+ does. The two automation templates are macOS launchd; on Linux use the equivalent cron lines (in the docs), on Windows use Task Scheduler.

What's on the roadmap?

Amazon Ads API campaign automation (CPR-sized ranking pushes — the auth client is already in scripts/), Search Query Performance funnel analysis, and multi-marketplace support. Issues and PRs welcome.
